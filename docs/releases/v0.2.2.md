# v0.2.2 — Day Detail & Weekly Review, Float data, refreshed enrichment UX

Status: draft (no ship date yet)

## What ships in v0.2.2

Three substantial additions land in this release.

### Day Detail & Weekly Review

Click any day on the Calendar to open a full **Day Detail Modal** — an at-a-glance view of that session without leaving the page:

- **Overview** — equity curve for the day plus a symbol-by-symbol breakdown.
- **Performance** — the day's metrics (P&L Ratio, win rate, and the rest) in one place.
- **Trades** — a day-scoped, sortable trade list; select a row to drill into the full Trade Detail.
- **Notes** — per-session notes with debounced autosave.
- **Mistakes** — a per-trade mistake rollup plus day-level tags.

Day Detail and Trade Detail stack cleanly — open a trade from inside a day, and Escape or a backdrop click closes only the top layer.

The same shell powers a new **Weekly Review Modal**: a Week Overview, a dense Week Performance table (day-by-day P&L, per-playbook breakdown, consistency), and matching Trades / Mistakes / Notes tabs for the week.

Both surfaces share a quick-glance **stat strip** so the numbers you check most are always at the top.

### Float data (arriving earlier than planned)

v0.2.1 deferred real float data to a later release. v0.2.2 brings it forward: FugaEdge now enriches trades with **float** sourced from Financial Modeling Prep (FMP), shown alongside shares outstanding as two distinct fields rather than one conflated number. When a provider has shares outstanding but no float for a symbol, FugaEdge says so plainly ("Unavailable") instead of guessing.

- New imports enrich automatically.
- A **backfill** action in Settings fills float for trades you imported before this release, touching only rows that are currently empty — it never overwrites data you already have.

### Refreshed enrichment UX

The market / intraday refresh flow got a thorough pass:

- A **live progress bar** that survives switching tabs mid-refresh.
- Refresh now defaults to **incremental**, with a shared **Force re-fetch** checkbox when you want a full refresh.
- A 15-second request timeout so a stalled fetch can't hang the whole refresh, plus a responsive cancel.
- Country detection now labels listing-inferred values as **"inferred"** rather than presenting them as confident, with a bulk per-symbol manual override.

## Migration

On first launch after upgrade, FugaEdge runs a one-time schema migration (20 → 21) to add the float fields and rename the prior shares-outstanding column to its accurate name. The migration is non-destructive: existing trade rows are not deleted or altered beyond the new/renamed columns being populated, and a backup is taken before it runs. If anything goes wrong, the backup is your recovery path (stop the app, restore it over `fugaedge.db`, delete the WAL/SHM sidecars, restart).

## Fixed

- Day Detail's session window now renders in Eastern time rather than UTC, matching the rest of the app's market-hours convention.
- The intraday refresh write-delta counters in Settings now read "…updated ·" so a zero clearly means "nothing newly changed," not "zero covered."

## What does NOT ship in v0.2.2

- **Per-date DST verification** on imported timestamps is still deferred — the current behavior (trust the literal timezone abbreviation in the file) is correct for in-season data. See `docs/plans/v0.3.0-or-later-ideas.md`.
- **Paper-account imports** remain gated off until the stats layer can keep paper trades out of real-account metrics (deferred to v0.3.0).
- **Per-trade collision actions** in the data-health banner remain deferred to a future Settings → Data Health page.
