# v0.2.3 — Delete Trade (soft-delete + restore + Trash)

Branch: `feature/v0.2.3-dave-feedback`.

Soft-delete a trade out of the active journal into a recoverable Trash, restore
it, or permanently purge it. Deleted trades drop out of every read path and
aggregate (stats, calendar, reports, analytics) but stay recoverable; re-
importing a deleted trade resurrects it. Distinct from the country-data
workstream that happens to share this branch — see `v0.2.3-dave-feedback.md`.

## Five-phase plan

**Phase 1 — schema + reads.** Add the `deleted_at` column and make every read
path soft-delete-aware, so deleted trades disappear from the journal, stats,
calendar, reports, and analytics by default while staying queryable.

**Phase 2 — repo lifecycle + import path + IPC.** The backend: lifecycle
operations (soft-delete / restore / hard-delete, single + bulk), resurrect-on-
reimport, daily-summary recompute for affected dates, and the IPC channels that
expose them to the renderer. Split into 2a (repo / import / recompute) and 2b
(IPC / preload bridge / attachment cleanup).

**Phase 3 — detail modal + confirm components.** Wire TradeDetailModal into the
lifecycle — soft-delete for live trades, restore for deleted ones — and build
the reusable confirm primitives behind it.

**Phase 4 — table bulk selection.** Multi-select rows in the Trades table to
soft-delete in bulk. (Restore is NOT here — it's a P5 Trash-card concern, since
the table only lists live rows; see the Q3 scope lock from P4 recon.) SHIPPED.

**Phase 5 — Settings Trash card.** The Trash surface: list deleted trades,
restore them, and permanently Delete Forever (type-to-confirm). Ships LAST.
PENDING.

## Locked decisions

1. Schema: `trades.deleted_at` TEXT nullable timestamp (NULL = live; ISO-8601 UTC = in Trash).
2. Re-import RESURRECTS: dedup query filters `deleted_at IS NULL` AND commit() issues an explicit un-delete UPDATE.
3. 30-day recovery window (UI copy only; auto-purge DEFERRED to a future release).
4. Trash UI = Settings → Trash card.
5. Confirm modals: soft-delete (preview + 30-day note); Delete-Forever (heavy, type-to-confirm). Restore = NO modal (instantly reversible).
6. getTrade(id) returns deleted rows (NO deleted_at filter); UI renders differently on deleted_at.
