## The blocker — do NOT re-litigate or re-guess
App hard-freezes on scroll. Fully diagnosed: an **async, self-sustaining ResizeObserver / lightweight-charts invalidation loop.**
- Evidence: `[LADDER-PERF]` logging showed `onRepaint` climbing unbounded with `reentrant=false` on all 843 calls, identical inputs every frame (`paneW=790 paneH=372 bars=798 markers=5`), sub-ms assemble.
- Therefore: data and layout are fine and fast. The loop is NOT an inline re-trigger — a reentrancy guard or dirty-check will NOT fix it (both already failed today).
- Mechanism: the signal primitive's `draw() -> onRepaint` re-enters via SCHEDULED invalidation a frame later. Almost certainly the SVG `applyFrame` write perturbs layout -> the chart container's ResizeObserver fires -> `chart.applyOptions({ width })` -> repaint -> `onRepaint` -> write -> repeat.

## The fix is STRUCTURAL, not a guard
Sever the ResizeObserver feedback. Two approaches, try (b) first (smaller):
- (a) Anchor the overlay SVG OUTSIDE the container the chart's ResizeObserver observes.
- (b) Gate the ResizeObserver handler so it only calls `applyOptions` when width ACTUALLY changed (skip no-op resizes).
