## 0. What already exists and ships (do NOT rebuild)

Banked on origin at `c77d219` + the local `3af23a2`:

- **Verdict stat block** — Ross's review order (net, avg daily, green-day %, largest red day) + 70%/2:1 reference gauges.
- **Edge Core / Consistency / Execution Quality / Behavior / Activity & Streaks** stat sections, A-vs-B with deltas + low-sample flagging.

**Locked design rules carried into v2 (do not violate):**
1. Gold = Period A, teal = Period B. Green/red = better/worse delta ONLY.
2. **No donut/pie/sunburst charts on analytical breakdowns** (rejected early; the reference "ring" images are mood, not literal targets).
3. No-fabricated-data law: null → em-dash, never 0/NaN. Thin samples flagged, not hidden. Coverage shown.
4. `src/core` stays Electron/DB/React-free (SaaS-port survival).
5. Each beat: recon → build → screenshot → commit. One topic per beat.

## 1. The honest scope assessment

This is **not "10% done"** — the data engine is largely built. But the *experience* is unfinished, which is the fair complaint. v2 splits into four work types, and the order matters because **type C may hide a data arc**:

| Type | What | Risk |
|---|---|---|
| **A. Styling/polish** | premium picker card, empty-space fill, card chrome, section polish | Low — renderer-only, hot-reloads |
| **B. New breakdowns, data EXISTS** | price bucket, float bucket (if stored), time-of-day quad | Medium — UI + maybe a core aggregator |
| **C. New breakdowns, data MISSING** | RVOL bucket, gap-% bucket | **High — may need data computed + stored first (own arc)** |
| **D. New chart types** | the TradesViz-style time-of-day quad, possibly per-bucket equity sparklines | Medium |

**The single most important unknown:** do `relative_volume`, `gap_pct`, and point-in-time `float` exist on the trade record today? Price-at-entry almost certainly does; float is enriched (probably yes); **RVOL and gap % may not exist at all.** If they don't, "add an RVOL breakdown" is a *data beat* (compute + store), not a UI beat. **A recon answers this before we commit to building those cards.** Everything in §3 marked ⚠️ is contingent on that recon.

## 2. Target layout (top → bottom)

The v2 Compare tab, in order. New/changed items marked **[NEW]** / **[RESTYLE]**.

1. **Period picker — premium card** **[RESTYLE]**
   - The top "Periods" card becomes the showpiece header. Period A (gold-accented) and Period B (teal-accented) as two premium cards with the felt-surface + subtle-glow treatment. Selected preset highlighted in the period's accent color. Date range, presets, shortcuts kept — re-skinned, not rebuilt. The "vs" framing made visually central.
   - *Why:* it's the first thing a user touches; right now it's plain. This sets the "real deal" tone immediately.

2. **Verdict stat block** *(exists — minor polish only)*
   - Keep as-is. Possibly tighten the gauge styling to match the new picker chrome. No structural change.

3. **The Activity & Streaks empty-space fix** **[NEW]**
   - Right now Activity & Streaks sits alone in the left column with dead space to its right (the odd card in a 2-col grid). Options (pick in §4): (a) a new card fills the gap, (b) Activity & Streaks goes full-width, (c) the new time-of-day quad lands there.
   - *My lean:* fill it with a genuinely useful card rather than stretch — strongest candidate is a **size-scaling / consistency widget** or the **time-of-day quad**, so the space earns its place.

4. **Cumulative P&L** *(exists)* + **R-multiple distribution** *(exists)* — keep.

5. **Time-of-Day quad** **[NEW]** *(the strongest reference idea — image 6)*
   - Four small panels — **PnL · Profit Factor · Accuracy · Trades** — bucketed by hour (or the existing 30-min buckets), Period A vs Period B. This is the TradesViz layout, rendered in our gold/teal idiom (NOT their blue/green). It's the densest, most momentum-relevant "when do I actually make money" cut, and it maps onto Ross's "first two hours" framing. **Data exists** (hour breakdown already computed) → this is type B, buildable without a data arc.

6. **Momentum breakdown cards** **[NEW]** — the float/price/RVOL/gap cuts, each a paired A-vs-B bar card like the existing seven:
   - **By Price bucket** ($1–5, $5–10, $10–20, $20+) — ⚠️ confirm price-at-entry stored (likely yes).
   - **By Float bucket** (<1M, 1–5M, 5–10M, 10–20M, 20M+) — ⚠️ confirm float stored (probably yes, it's enriched).
   - **By Relative Volume bucket** (<5×, 5–10×, 10×+) — ⚠️ **likely MISSING data — own arc.**
   - **By Gap % bucket** (<10%, 10–30%, 30–50%, 50%+) — ⚠️ **likely MISSING data — own arc.**
   - *Why these matter:* they ARE Ross Cameron's scan pillars (float <20M, RVOL ≥5×, price $2–20, gap ≥10%). No competitor journal compares two periods by float/RVOL. This is the literal "what a momentum trader needs" — and the flagship wedge.

7. **The existing 7 breakdown cards** *(exist)* — kept, possibly restyled to match the new card chrome so the whole grid is consistent.

8. **Comparison Insights** *(exists, honesty-fixed)* — keep.

**What we are NOT adding (and why):** donut/ring gauges for win/loss rate (images 3,4,5,7) and the symbol sunburst (image 8) — they violate the no-donut rule and the green/red-for-deltas rule. We take their *polish* (depth, glow, premium dark) into the card chrome, not their chart types.

## 6. The honest caveats (so we don't overpromise the masterpiece)

- **"All sorts of graphs" has a ceiling worth respecting.** More charts ≠ more premium. The reference images that look best (image 6) are *dense and scannable*, not numerous. The masterpiece is the *right* graphs done beautifully, not every graph. I'll push back if a beat starts adding charts that don't earn their space.
- **Type C is a real fork.** If RVOL/gap data doesn't exist, building it touches the import pipeline + a backfill + possibly Polygon calls — that's a multi-day arc on its own, comparable to a whole feature. We decide deliberately, not mid-stream.
- **The no-donut / gold-teal / no-fabrication rules are not negotiable for aesthetics.** The reference rings are inspiration for *polish*, not license to break the design system that makes FugaEdge coherent.
- **This is 6–10 beats, not one.** Pacing it is what keeps it premium. A masterpiece gets built brick by verified brick.
