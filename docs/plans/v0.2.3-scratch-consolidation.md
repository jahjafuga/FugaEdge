# v0.2.3 — Scratch-Trade Consolidation (single source of truth)

Branch: `feature/v0.2.3-dave-feedback`. Bundled into the v0.2.3 release alongside
the delete-trade / Trash work (`v0.2.3-delete-trade.md`).

## The bug (Dave, DTSM DMs)

"Scratch" was defined inconsistently across the app — two definitions had
coexisted since v0.1.0:

- **±$2 band** (`|net_pnl| ≤ $2` → scratch): Reports Breakdown, Tier Performance,
  Playbook / Setup-Library, Performance comparison, Today's Session.
- **Strict-zero / bare-sign** (`net_pnl > 0` / `< 0`): Daily Detail, Weekly
  Review, Dashboard, Calendar, Analytics, Edge Insights, Trades page.

The v0.2.2 calendar / day-detail work added the strict-zero surfaces, so the SAME
day rendered different W/L/SCR splits depending on which surface you looked at.
Dave's Jun 1 screenshots: Daily Detail showed 5W/5L/2SCR (strict) while Reports
Breakdown showed 5W/3L/4SCR (the ±$2 band reclassified 2 real losses as scratch);
Tier Performance's scratch column was inflated for the same reason.

## Locked decision

Kill the ±$2 band. **Scratch = `|net_pnl| ≤ SCRATCH_EPSILON` (0.005)** — strict
zero with a float-representation tolerance (`net_pnl` is a REAL column; 0.005
rounds to $0.00 at 2-decimal display). ONE definition, applied at every per-trade
classification site through a shared helper. Aggregate "is this day/week/bucket
net positive?" sign-checks stay bare-sign (decision R2a — a different axis;
deferred to v0.3.0, see the parking lot).

## Retroactive stat shift (upgrade note)

Users upgrading from v0.2.2 will see **Reports / Tier Performance / Playbook /
Today's-Session** numbers — win rate, profit factor, avg winner/loser, expectancy,
hold-time-by-outcome, max-consecutive — shift slightly: near-breakeven trades
formerly inside the ±$2 band (e.g. a −$1.50 trade) now count as the real small
wins/losses they are instead of scratch. The numbers CONVERGE toward the Dashboard
and Daily-Detail surfaces, which were already on strict-zero. This is the intended
fix — Dave's "real losses got hidden" complaint. **No trade data is changed**:
`trades` is untouched; only the derived `daily_summary` cache is rebuilt, and the
live surfaces recompute on read.

## Release-notes stub (Lao to finalize at release time)

> **v0.2.3**
>
> - **Trash (delete a trade)** — soft-delete trades out of the active journal into
>   a recoverable Trash (Settings → Trash), restore them, or Delete Forever.
>   30-day recovery window shown in the UI.
> - **Scratch-trade fix** — "scratch" now means one consistent thing on every
>   screen: a trade that netted essentially $0.00 (a sub-cent tolerance). This
>   replaces an old ±$2 rule that some screens used and others didn't, which made
>   the same day show different win/loss/scratch counts in different places.
>   **Heads-up:** if your win rate or profit factor shifts a little after
>   updating, that's expected — near-breakeven trades that some screens were
>   hiding as "scratch" now count as the real small wins/losses they always were.
>   Your trade data is unchanged.
